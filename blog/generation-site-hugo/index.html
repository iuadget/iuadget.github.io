Генерация сайта на HUGO

Опубликовано 2017-05-28

За прошедший год идей для статей было много, но ни одна из них не превратился в статью.
За это время была переустановлена система и все исходники блога были удалены, осталось только сгенерированная статика.
Чтоб не забыть, небольшие заметки:

1) Установить Hugo.
Для этого воспользуемся Homebrew.
Пишем следующую команду в терминале:

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

После успешной инсталляции приступаем к установке Hugo, для этого в терминале пишем:

brew install hugo

Hugo установлен.

2) Создать сайт.
Воспользуемся статьей Hugo - Hugo Quickstart Guide
2.1) Создаем новый сайт. В терминале пишем следующую строку:

hugo new site best-site

Сайт успешно создан.
2.2) Выбираем тему.
Для созданных сайтов, каталог с темами пустой, поэтому придется выбрать и установить тему сайта. Все темы располагаются в каталоге themes. Перейдем в каталог с темами:

cd themes

Клонируем тему в каталог.

git clone https://github.com/dim0627/hugo_theme_robust.git

Тема успешно клонирована.

Большой выбор тем для сайта доступен в Каталоге тем для Hugo

2.3) Настраиваем конфигурационный файл.
В корне нашего сайта лежит конфигурационный файл config.toml. По умолчанию файл конфигурации использует Формат TOML, но также можно использовать формат YAML или JSON. Параметры конфигурации, указанные в config.toml применяются к полной версии сайта.

name = "Robust"
license = "MIT"
licenselink = "https://github.com/dim0627/hugo_theme_robust/blob/master/LICENSE.md"
description = "Grid based theme for Hugo."
homepage = "https://github.com/dim0627/hugo_theme_robust"
tags = ["blog"]
features = ["responsive", "thumbnail", "grid", "syntax highlight", "structured data", "ogp", "twitter cards"]
min_version = 0.20.2

[author]
  name = "Daisuke Tsuji"
  homepage = "https://yet.unresolved.xyz/"

Добавим следующую информацию:

baseURL = "http://наш-сайт.рф/"
languageCode = "ru-RU"
title = "The Best Site"

[Params]
  Author = "Самый лучший писатель"

2.4) Создаем статью.
Для того чтобы создать новую статью, напишем следующее:

hugo new post/good-to-great.md

В результате в каталоге content должен появиться файл в формате Markdown:

content
`-- post
    `-- good-to-great.md

В любом удобном редакторе начинаем писать статью.
Для проверки отображения сайта и публикуемой статьи запустим сервер:

hugo server --theme=hugo_theme_robust --buildDrafts

Чтобы просмотреть что получилось, переходим по адресу http://localhost:1313/.

2.5) Генерация статичного сайта для публикации.
Когда все написано и проверено необходимо сгенерировать сайт, для этого пишем:

hugo --theme=hugo_theme_robust

После выполнения команды в корне сайта будет создан каталог public, содержащий сгенерированный исходный сайт.

3) Размешаем сайт.
Хостим полученный сайт, например на Github Pages.
